# EEGLAB-DAG Pipeline Editor

A node-based pipeline editor for EEGLAB, allowing users to construct, visualize, and execute EEG processing workflows.

## Features

- **Visual Interface**: Drag-and-drop nodes to create processing pipelines.
- **EEGLAB Integration**: Runs directly from the EEGLAB menu.
- **Community Driven**: Easily share custom node definitions and pipelines.
- **Cross-Platform**: Works on Windows, macOS, and Linux (Standalone executables provided).

## Installation

### For Users (No Python Required)

1.  Download the latest **Release ZIP** from the [GitHub Releases](https://github.com/sccn/eeglab-dag/releases) page.
    *   *Note: Do not download the "Source code" zip. Use the release asset (e.g., `EEGLAB-DAG-v1.0.zip`).*
2.  Unzip the folder into your EEGLAB plugins directory:
    *   `eeglab/plugins/EEGLAB-DAG`
3.  Restart MATLAB/EEGLAB.

### For Developers (Run from Source)

1.  Clone the repository:
    ```bash
    git clone https://github.com/sccn/eeglab-dag.git
    ```
2.  Install Python dependencies:
    ```bash
    pip install -r requirements.txt
    ```
3.  Configure MATLAB to find your Python environment (if not automatically detected).

## Usage

1.  Start EEGLAB (`eeglab`).
2.  Navigate to **DAG > DAG Editor**.
3.  **Build your pipeline**:
    *   Right-click on the canvas to add nodes.
    *   Connect nodes to define the data flow.
    *   Select a node to edit its parameters in the sidebar.
4.  **Save/Load**: Use the toolbar to save your workflow. By default, pipelines are saved to `library/pipelines/`.
5.  **Run**: Click the "Run" button to execute the pipeline.

## Community & Customization

The project is designed to be extensible.

### Adding New Nodes
The node definitions are stored as JSON files in the `library/nodes/` folder. To add a new EEGLAB function:
1.  Create a JSON file describing the function (inputs, outputs, parameters).
2.  Place it in `EEGLAB-DAG/library/nodes/`.
3.  Restart the DAG Editor.

### Sharing Pipelines
Pipelines are saved as JSON files in `EEGLAB-DAG/library/pipelines/`. You can share these files with others.

## Development

### Directory Structure

```
EEGLAB-DAG/
├── bin/                  # Pre-compiled binaries for each OS (generated by build script)
├── library/
│   ├── nodes/            # JSON definitions for EEGLAB functions
│   └── pipelines/        # Saved pipeline files
├── src/                  # Source code (Python & MATLAB)
│   ├── gui/              # PyQt6 GUI components
│   ├── model/            # Data models and logic
│   └── matlab/           # MATLAB integration scripts
├── build_dist.py         # Script to build standalone executables
└── requirements.txt      # Python dependencies
```

### Building Standalone Executable
To build the frozen binaries for your OS:
```bash
python build_dist.py
```
This will generate the executable in `bin/<os>/main/`.
